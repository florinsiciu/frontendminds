## 1. Migration & Version Health <!-- Fraunces, dimension color: #93C5FD -->

*Stay current to access security patches, performance gains, and ecosystem support.*

- [ ] Upgrade to the latest stable Angular version (check angular.dev for current LTS)
- [ ] Remove all AngularJS remnants — no hybrid bootstrap, no ngUpgrade bridges
- [ ] Audit third-party dependencies — flag anything unmaintained or pinned to old Angular
- [ ] Establish a quarterly upgrade cadence — never fall more than 1 major version behind

---

## 2. Codebase Architecture <!-- Fraunces, dimension color: #5EEAD4 -->

*Clean boundaries prevent cascading failures and make onboarding faster.*

- [ ] Adopt a monorepo or feature-boundary structure with clear module ownership
- [ ] Document and enforce architectural standards (ADRs, linting rules, PR templates)
- [ ] Schedule an annual codebase architecture review — assess structure, boundaries, and debt
- [ ] Ensure feature modules can be developed and tested independently — no hidden cross-dependencies

---

## 3. Modern Angular Adoption <!-- Fraunces, dimension color: #FBBF24 -->

*Standalone components, Signals, and modern patterns are the future — start migrating now.*

- [ ] Migrate to standalone components — eliminate NgModules from new code immediately
- [ ] Define a consistent reactive state strategy — Signals, RxJS, or a clear hybrid approach
- [ ] Simplify forms — reduce boilerplate with typed Reactive Forms, CVA patterns, or form builders
- [ ] Build a test suite your team trusts enough to refactor with confidence

---

## 4. AI & Development Governance <!-- Fraunces, dimension color: #C4B5FD -->

*AI accelerates teams that have guardrails — and creates chaos for those that don't.*

- [ ] Publish AI coding guidelines — what tools are approved, what patterns to follow
- [ ] Ensure code review standards apply equally to AI-generated and human-written code
- [ ] Configure AI tools to follow your project's architectural patterns and conventions
- [ ] Use AI to accelerate modernization — migration scripts, test generation, documentation

---

## 5. Delivery & Talent Readiness <!-- Fraunces, dimension color: #F87171 -->

*Modern code means nothing if you can't ship it or staff it.*

- [ ] Reduce time-to-hire for Angular roles — modern stacks attract better candidates faster
- [ ] Build deployment confidence — every merge to main should be deployable without fear
- [ ] Identify and remove legacy Angular patterns that block your team from shipping new features
- [ ] Align leadership on a modernization timeline with clear milestones and budget
